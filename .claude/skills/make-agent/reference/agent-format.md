# 서브에이전트 정의(.claude/agents/*.md) 정본 스펙

make-agent(author)와 make-agent-reviewer가 공유하는 단일 기준 문서. 필드 스펙과 본문 구조는 여기를 따른다.

## frontmatter

### 필수
| 필드 | 의미 |
| --- | --- |
| `name` | Agent의 고유 이름. **파일명(`<name>.md`)과 반드시 일치.** |
| `description` | 언제 이 Agent를 호출해야 하는지 설명. 오케스트레이터가 호출 여부를 판단하는 근거 = **호출 트리거.** |

### 선택
| 필드 | 의미 |
| --- | --- |
| `tools` | 사용할 수 있는 도구 허용 목록 |
| `disallowedTools` | 사용하지 못하게 할 도구 목록 |
| `model` | Agent가 사용할 모델 |
| `permissionMode` | 권한 승인 방식 |
| `maxTurns` | 최대 Agent 실행 턴 수 |
| `skills` | 시작할 때 미리 불러올 Skill |
| `memory` | Agent 전용 지속 메모리 범위 |
| `background` | 백그라운드 실행 여부 |
| `effort` | 추론 노력 수준 |
| `isolation` | 별도 Git worktree에서 실행할지 |
| `color` | UI에 표시되는 Agent 색상 |
| `mcpServers` | Agent가 사용할 MCP 서버 |
| `hooks` | Agent 실행 생명주기 Hook |
| `initialPrompt` | 메인 Agent로 실행될 때 최초 입력 |

### 필드 결정 원칙
- **name, description — 언제 호출될지 결정.** name은 파일명과 일치. description은 "구체적으로 무엇을 다루는지"를 적어 호출 트리거 역할을 하게 한다. **동작 방식(HOW)이 아니라 호출 조건(WHEN)을 쓴다.**
- **model — 추론 깊이에 따라 결정.** 비용·모델 등급·응답 속도를 한꺼번에 결정한다. (무거운 추론=opus, 일반=sonnet, 단순·대량=haiku)
- **tools — 부작용 범위에 따라 결정.** 허용 도구 목록. `disallowedTools`가 **먼저** 적용되어 상속 목록에서 해당 도구를 제거하고, **남은 집합에 `tools` 필터**가 적용된다. 읽기/조사 전용이면 Write/Edit를 넣지 않는다(최소 권한).

```markdown
---
name:
description:
---
```

## 본문 구조

목적별 섹션. **필수** 표시는 항상, 조건부는 해당 상황에서만 넣는다(불필요한 섹션을 억지로 채우지 말 것).

### 핵심 역할 — 범위 결정 (필수)
하는 일을 1~3개로 명시한다. '무엇을 하는가'와 '무엇을 다루는가'를 명확히 밝힌다.

### 작업 원칙 — 판단 기준 결정 (필수)
두 선택지가 충돌할 때 어느 쪽을 택할지 정한다.

### 입출력 프로토콜 — 에이전트 간 API 계약 (다른 에이전트와 산출물을 주고받을 때)
다른 에이전트가 이 에이전트의 산출물을 읽을 경로와 형식을 정한다. 입력·출력·형식을 명시한다.

### 팀 통신 프로토콜 — 누구에게·무엇을·언제 (팀 모드에서만)
`SendMessage` 도구로 주고받을 정보의 세 요소(누구에게·무엇을·언제)를 명시한다.
오케스트레이터가 `TeamCreate`로 팀을 구성하는 팀 모드에서만 의미가 있다.

### 에러 핸들링 — 종료 조건 명시 (실패할 수 있는 작업이면)
구체적 실패 시나리오, 각 시나리오에서의 행동, 최대 재시도 횟수를 정한다.

### 협업 — 팀 안에서의 위치 결정 (파이프라인에 속할 때)
누구와 어떤 순서로 연결되는지 명시한다. 상류=나를 부르는 쪽, 하류=내가 결과를 넘기는 쪽, 재호출 조건.

### 품질 자체 검증 — 제출 전 자기 점검 (권장)
에이전트가 스스로 거쳐야 할 체크리스트를 둔다(해당 작업 고유의 검증 항목).
